## 🧩 OPEN-001 – Why OpenAPI?

### 🎯 Learning Goal

Understand the **purpose and value** of OpenAPI / Swagger in modern software development.
You should be able to explain **what problems OpenAPI solves**, **who benefits from it**, and **why standardized API documentation matters**.

Additionally, you should be able to relate these concepts to a **real API example** provided in this project.

---

### 📝 Task

1. **Research** with the help of ChatGPT or your preferred AI assistant:

   * What is OpenAPI?
   * What are the key use cases?
   * Why is standardized API documentation important?
   * What is the difference between **OpenAPI (specification)** and **Swagger (tooling)**?

2. **Explore** official sources such as:

   * [OpenAPI Specification](https://swagger.io/specification/)
   * [Swagger.io Overview](https://swagger.io/tools/)
   * [Stoplight Blog](https://stoplight.io/openapi/)

3. **Work with the provided project**

   * Start the API (`npm install` → `npm start`)
   * Open Swagger UI:
     👉 [http://localhost:3000/docs](http://localhost:3000/docs)
   * Explore the documented endpoints

4. **Reflect on the following guiding questions:**

   * What challenges do teams face when APIs are undocumented or inconsistently documented?
   * How does OpenAPI improve collaboration between frontend, backend, and third-party developers?
   * What advantages do automated tools (e.g. Swagger UI, code generators, mock servers) offer?
   * What would be harder if this project only contained `server.js` but no `openapi.yaml`?

5. **Write a short summary** (4–6 sentences) answering:

   > “What is OpenAPI, which problems does it solve, and how does it improve the quality and usability of APIs?”

   Your summary should include:

   * A short definition of OpenAPI
   * At least **3 concrete benefits**
   * A short comparison: OpenAPI vs. no documentation
   * A reference to this project (Swagger UI or `openapi.yaml`)

📌 Save your summary in a file (e.g. `notes.md`) or prepare it for discussion.

---
